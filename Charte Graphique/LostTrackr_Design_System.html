6. Règles & Conventions
Couleurs
Utiliser les variables CSS
Toujours référencer var(--blue), var(--text), var(--green)… Ne jamais répéter la valeur hex directement dans une nouvelle règle CSS.
Inventer de nouvelles couleurs
L'app est entièrement dark. Pas de blanc pur, pas de gris clair, pas de fond clair. Toute nouvelle couleur doit appartenir à la famille bleue nuit ou aux accents existants.
Opacité basse sur les bordures
Les bordures sont en rgba() avec opacité 0.12 à 0.28 maximum. Les bordures opaques cassent la profondeur visuelle de l'app.
Ne pas intervertir les couleurs sémantiques
--green = succès confirmé uniquement. --orange = attention / à vérifier. --danger = erreur critique / action destructive. Usage strict.
Typographie
Respecter l'échelle définie
Titres de page : 49px / 800. Titres de cards : 28-32px / 700-800. Corps : 18-22px / 400. Labels section : 11px uppercase. Ne pas créer de taille intermédiaire non listée.
Chemins de fichiers en monospace
Tout texte représentant un chemin système : balise <code> avec font-family monospace et couleur #b8c6d8.
Pas de text-transform uppercase sur les titres
Uppercase réservé aux labels de section (10-11px, letter-spacing .1em). Pas sur les titres de page, de card ou de bouton.
Composants
Toujours inset white-line sur les cards
Toute nouvelle card : box-shadow: inset 0 1px 0 rgba(255,255,255,.05) pour simuler un reflet en haut et renforcer la profondeur.
Pills pour les états, boutons pour les actions
Les state-pill décrivent un état (OK, Inclus, À vérifier). Pour déclencher une action, utiliser les boutons primary / secondary / tertiary.
Pas de backdrop-filter sur de nouveaux éléments
Le modal et l'onboarding utilisent backdrop-filter: blur(16px). Ne pas ajouter ce filtre ailleurs — trop coûteux en perf sur les vieilles machines des DJs.
Radial glow pour chaque nouvelle vue
Chaque grande section a un radial-gradient de glow bleu en arrière-plan. Pour une nouvelle vue : radial-gradient(Xpx Ypx at 50% 26%, rgba(10,76,145,.11), transparent 70%).
Motion
Toujours respecter prefers-reduced-motion
La règle @media (prefers-reduced-motion: reduce) désactive tout. Chaque nouvelle animation doit automatiquement en faire partie — pas d'exception.
Pas d'animation sur les éléments de données
Les paths, noms de fichiers, compteurs — pas d'animation (sauf rollTo sur les chiffres déjà en place). Animer uniquement les éléments décoratifs et les transitions de vue.
Durées : micro 0.16s · UI 0.18s · décoratif 3.8s+
Micro-interactions (boutons, hover) : 0.16-0.18s. Transitions de vue : 0.2-0.3s. Animations radar / idle : 3.8 secondes minimum.
