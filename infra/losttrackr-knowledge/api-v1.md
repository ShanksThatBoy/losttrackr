# LostTrackr Knowledge API v1

The desktop app must never connect directly to PostgreSQL. All communication
goes through HTTPS endpoints exposed through Cloudflare Tunnel.

## Base URLs

Production:

```text
https://knowledge.losttrackr.com/v1
```

Staging:

```text
https://knowledge-staging.losttrackr.com/v1
```

## Authentication

All catalog, sync, and contribution endpoints require a Pro app session token
issued by the separate license service.

```http
Authorization: Bearer <app_session_token>
X-LostTrackr-App-Version: 1.2.0
X-LostTrackr-Platform: macos | windows
X-LostTrackr-Install-Id: <random uuid generated by the app>
```

`X-LostTrackr-Install-Id` must be a random app identifier. It must not contain a
machine name, OS username, device serial number, or email address.

## Health

```http
GET /health
```

Returns API and dependency status for the desktop app and monitoring.

```json
{
  "status": "ok",
  "database": "ok",
  "redis": "ok",
  "catalog_snapshot_at": "2026-06-30T08:00:00Z"
}
```

```http
GET /catalog/search?q=young%20thug&limit=10
```

manual correction UI.

## Match Local Tracks

```http
POST /catalog/match
```

Used by LostTrackr after a scan to identify local tracks against the knowledge
base. This endpoint accepts cleaned metadata only.

Request:

```json
{
  "tracks": [
    {
      "client_track_id": "local-001",
      "title": "Digits",
      "artist": "Young Thug",
      "album": "Slime Season 3",
      "duration_ms": 176000,
      "bpm": 130.0,
      "camelot_key": "8A",
      "genre": "Hip-Hop",
      "isrc": "USAT21601032",
      "source_app": "serato"
    }
  ]
}
```

Response:

```json
{
  "matches": [
    {
      "client_track_id": "local-001",
      "status": "matched",
      "recording_id": "8a4e0b40-2f03-447f-9784-fcf4b3a71528",
      "confidence": 0.96,
      "canonical": {
        "title": "Digits",
        "artist": "Young Thug",
        "duration_ms": 176000,
        "bpm": 130.0,
        "camelot_key": "8A",
        "genre": "Hip-Hop"
      }
    }
  ]
}
```

## Submit Opt-In Contributions

```http
POST /contributions/batches
```

Used only after the user explicitly validates contribution from the Pro feature
UI. The app must apply a privacy filter before sending data.

Request:

```json
{
  "consent_version": "knowledge-contribution-fr-2026-06-30",
  "source_app": "serato",
  "app_version": "1.2.0",
  "platform": "macos",
  "privacy_filter_version": "1",
  "items": [
    {
      "title": "Digits",
      "artist": "Young Thug",
      "album": "Slime Season 3",
      "remix": null,
      "duration_ms": 176000,
      "bpm": 130.0,
      "musical_key": "A minor",
      "camelot_key": "8A",
      "genre": "Hip-Hop",
      "year": 2016,
      "isrc": "USAT21601032",
      "source_confidence": 0.82
    }
  ]
}
```

Forbidden fields:

- local file path;
- folder name;
- crate or playlist name;
- user comment;
- play history;
- machine name;
- email;
- raw DJ software file contents;
- audio file or audio excerpt.

Response:

```json
{
  "batch_id": "cf0cfbf8-3684-40f9-9416-0ea14b7e74211",
  "accepted": 1,
  "rejected": 0,
  "retention_until": "2026-12-30T00:00:00Z"
}
```

## Sync Offline Knowledge Package

```http
GET /sync/packages/latest?channel=beta&platform=macos
```

Returns the latest compact offline package available to the app. The package is
used when the API is unavailable.

```json
{
  "package_version": "2026.06.30.1",
  "min_app_version": "1.2.0",
  "catalog_snapshot_at": "2026-06-30T08:00:00Z",
  "download_url": "https://knowledge.losttrackr.com/v1/sync/packages/...",
  "sha256": "9b8f...",
  "size_bytes": 52428800
}
```

## Error Model

Every error response must use the same shape.

```json
{
  "error": {
    "code": "knowledge_unavailable",
    "message": "LostTrackr n'arrive pas à joindre le centre de connaissances.",
    "retryable": true
  }
}
```

Initial error codes:

- `unauthorized`
- `subscription_required`
- `contribution_consent_required`
- `invalid_payload`
- `rate_limited`
- `knowledge_unavailable`
- `maintenance`

## Desktop Offline Behavior

If the API is unavailable:

1. LostTrackr shows a clear sync status message.
2. Pro scan still works with the last synced package.
3. Contributions are queued locally only if the user previously accepted.
4. Queued contributions are sent during the next successful scan sync.
